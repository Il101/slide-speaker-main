# Исправление позиционирования визуальных эффектов

## Проблема
При нажатии кнопки Edit визуальные эффекты (highlight boxes) отображались в неправильных позициях - все элементы были "съехавшими".

## Причина
В `Canvas2DRenderer.getEffectBBox()` координаты bbox трансформировались с использованием **как масштаба, так и offset изображения**:

```typescript
const transformed = [
  x * scale.x + offset.x,  // ❌ Неправильно!
  y * scale.y + offset.y,  // ❌ Неправильно!
  w * scale.x,
  h * scale.y
];
```

Проблема в том, что:
1. **Изображение слайда** имеет CSS transform с масштабом и offset для центрирования
2. **Canvas для визуальных эффектов** занимает **весь контейнер** (`absolute inset-0`) БЕЗ transform
3. Координаты bbox из OCR даны в пикселях оригинального слайда (например, 1920x1080)
4. Canvas должен трансформировать координаты ТОЛЬКО с масштабом, БЕЗ offset

## Решение
Изменен метод `getEffectBBox()` в `Canvas2DRenderer.ts`:

```typescript
// 🔥 FIX: Canvas занимает весь контейнер БЕЗ offset!
// Применяем только масштаб, БЕЗ offset изображения
const scale = this.config.slideScale ?? { x: 1, y: 1 };

const transformed = [
  x * scale.x,  // ✅ Только масштаб
  y * scale.y,  // ✅ Только масштаб
  w * scale.x,
  h * scale.y
];
```

## Как проверить
1. Откройте урок в браузере
2. Нажмите кнопку "Edit Edit"
3. Визуальные эффекты должны точно совпадать с соответствующими элементами на слайде

### Debug в консоли браузера
Скопируйте содержимое `debug_vfx_positioning.js` в консоль DevTools для проверки:
- Размеры контейнера, изображения и canvas
- Вычисленный масштаб
- Пример трансформации координат

## Техническая справка

### Координатные системы
1. **Оригинальные координаты слайда**: Координаты из OCR в пикселях исходного слайда (например, 1920x1080)
2. **Координаты контейнера**: Размер контейнера в браузере (например, 800x450)
3. **Масштаб**: `containerSize / originalSlideSize`

### Трансформация для разных элементов
- **Изображение слайда**: Использует `transform: translate(offset) scale(scale)` для центрирования
- **Canvas визуальных эффектов**: Использует только `scale`, т.к. уже занимает весь контейнер
- **HTML оверлеи (в старом Player.tsx)**: Используют `scale` и `offset` для абсолютного позиционирования
