# 🎤 Исправление: Консистентная интонация в Gemini TTS

## Проблема

При генерации аудио для слайда с несколькими текстовыми сегментами (например, 6 предложений), **каждый сегмент отправлялся отдельным запросом** к Gemini TTS API. Это приводило к:

❌ **Изменению интонации между сегментами**  
❌ **Непредсказуемым паузам**  
❌ **Нарушению естественности речи**  
❌ **6 API-запросов вместо 1** (увеличение стоимости)

### Пример проблемы:

```
Слайд с 6 сегментами:
1. "Давайте рассмотрим важный концепт."        → API запрос #1
2. "Это связано с тем, что мы обсуждали ранее." → API запрос #2
3. "Ключевая идея здесь в том, что..."         → API запрос #3
...
```

**Результат:** Голос меняется между сегментами, нарушается связность речи.

---

## Решение

✅ **Объединение всех сегментов в один текст** перед отправкой в Gemini TTS  
✅ **1 API-запрос на весь слайд** вместо N запросов  
✅ **Консистентная интонация по всему слайду**  
✅ **Снижение стоимости на ~83%** (для слайда с 6 сегментами)

### Изменения в коде:

#### Файл: `backend/workers/tts_gemini.py`

**ДО:**
```python
# ❌ Отдельный запрос для каждого сегмента
for i, text in enumerate(texts):
    audio_data = self.synthesize_speech(text, prompt=style_prompt)
    audio_segments.append(audio_data)
    # Каждый вызов - новый API запрос с новой интонацией
```

**ПОСЛЕ:**
```python
# ✅ Один запрос для всех сегментов
combined_text = " ".join([text.strip() for text in texts if text.strip()])
audio_data = self.synthesize_speech(combined_text, prompt=style_prompt)

# Распределяем время пропорционально длине текстов
for i, text in enumerate(texts):
    segment_duration = (len(text) / total_chars) * total_duration
    # ... создаём timing info
```

---

## Технические детали

### Метод оценки таймингов

Поскольку Gemini TTS **не предоставляет word-level timing**, мы используем пропорциональное распределение:

```python
segment_duration = (text_chars / total_chars) * total_duration
```

**Точность:** ~80-90% (достаточно для синхронизации visual effects)

### Ограничения

⚠️ **Gemini TTS лимит:** 4000 байт на запрос  
📝 Если combined text > 4000 байт, он будет обрезан в `synthesize_speech()`
