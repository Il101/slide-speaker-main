## 🔧 ДОПОЛНИТЕЛЬНОЕ ИСПРАВЛЕНИЕ #2

### Проблема: Неправильное использование [visual:XX]

**Обнаружено:** ИИ использовал `[visual:XX]VU - 717025[/visual]` для текста, который ДОЛЖЕН произноситься  
**Результат:** Текст появлялся в субтитрах и произносился (неправильно)

**Ожидаемое поведение [visual:XX]:**
- ❌ НЕ произносится (TTS делает паузу)
- ❌ НЕ в субтитрах (скрыт)
- ✅ Только визуальные эффекты (подсветка)

### Исправления в промпте (строки 448-506):

**1. Явное предупреждение:**
```
🚨 ВАЖНО: Используй [visual:XX] ТОЛЬКО для текста, который НЕ нужно произносить!

КОГДА ИСПОЛЬЗОВАТЬ [visual:XX]:
✅ Длинные иностранные названия, которые сложно произнести
✅ Номера курсов, коды (VU-717025)
✅ Сложные латинские названия
❌ НЕ для обычного текста в субтитрах!
❌ НЕ для терминов, которые нужно произнести!
```

**2. Добавлена Option 3 - Обычный текст (БЕЗ маркеров):**
```
**Option 3: Обычный текст (БЕЗ маркеров)**
- ❌ НЕ используй маркеры для обычного текста!
- ✅ Просто пиши текст как есть
- TTS произносит всё как написано
- Всё показывается в субтитрах
```

**3. Чёткая матрица выбора:**
```
🎯 ВЫБИРАЙ ПРАВИЛЬНО:
1. [visual:XX] → Длинный иностранный термин/код, который НЕ произносим
2. [lang:XX] → Короткий иностранный термин, который произносим
3. Обычный текст → Всё остальное (русские слова, объяснения)
```

**Примеры:**
```
✅ "Также указан код курса [visual:en]VU - 717025[/visual] для идентификации"
   → TTS: "Также указан код курса [pause] для идентификации"
   → Subtitles: "Также указан код курса для идентификации"
   → Visual: highlights "VU - 717025"

✅ "Рассмотрим типы листьев: простые, сложные, чешуйчатые"
   → TTS: произносит всё
   → Subtitles: всё отображается
   → Visual: highlights text on slide
```

## 🔧 ДОПОЛНИТЕЛЬНОЕ ИСПРАВЛЕНИЕ #3

### Проблема: Чрезмерный разбор названий курсов

**Обнаружено:** ИИ тратит время на детальный разбор заголовков:
```
"Итак, группа номер один – это название курса: AUFBAU UND ARCHITEKTUR der 
PflanZEN KURS 3 : Zellwand, Osmo. Это означает, что мы будем изучать строение 
и архитектуру растений, и это третий курс в серии, посвященный клеточной 
стенке и осмосу."
```

**Результат:** 
- 🔴 Тратится 10-15 секунд на анализ названия
- 🔴 Студенты не получают полезной информации
- 🔴 Меньше времени на реальный контент

**Ожидаемое поведение:** Заголовок нужно просто НАЗВАТЬ (1 предложение), не разбирать

---

### Исправления

#### 1. Снижен вес title в адаптивном промпте

**Файл:** `backend/app/services/adaptive_prompt_builder.py:30`

**Было:**
```python
TYPE_WEIGHTS = {
    'title': 3.0,  # Высокий вес → ИИ считает важным
    ...
}
```

**Стало:**
```python
TYPE_WEIGHTS = {
    'title': 1.5,  # ✅ FIX: Снижен вес - заголовки не требуют детального разбора
    ...
}
```

**Влияние:**
- Title теперь менее приоритетен, чем heading (2.5) и key_point (2.0)
- AdaptivePromptBuilder будет ранжировать контент выше заголовков
- Заголовки попадут в секцию 'MENTION' (просто упомянуть), не 'DETAILED'

---

#### 2. Добавлена специальная инструкция для заголовков

**Файл:** `backend/app/services/smart_script_generator.py:442-448`

**Добавлено в промпт:**
```
🚨 SPECIAL RULE FOR TITLES/COURSE NAMES:
❌ DO NOT analyze or explain titles/course names in detail!
✅ Simply MENTION the title briefly (1 sentence max):
   "Сегодня тема: [название]" OR "Это курс номер X о [тема]"
❌ WRONG: "Итак, группа номер один – это название курса... Это означает, что мы будем изучать..."
✅ CORRECT: "Сегодня курс номер 3 о клеточной стенке и осмосе"
```

**Эффект:**
- Явный запрет детального анализа заголовков
