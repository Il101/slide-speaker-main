# Упрощенный индикатор прогресса обработки

## Что было изменено

Создан новый упрощенный компонент `SimpleProcessingProgress.tsx` вместо сложного `ProcessingProgress.tsx`.

## Новый дизайн

### Внешний вид:
```
┌─────────────────────────────────────────────┐
│         ⚙️ Обработка презентации            │
│                                             │
│  ████████████████░░░░░░░░░░░  75%          │
│                                             │
│  Генерация текста лекции • Шаг 3 из 5      │
└─────────────────────────────────────────────┘
```

### Элементы:
1. **Заголовок с иконкой** - показывает статус (обработка/ошибка)
2. **Прогресс-бар** - визуальное отображение прогресса (0-100%)
3. **Текстовая информация** - название текущего шага и номер шага

## Преимущества нового подхода

### ✅ Простота
- Один прогресс-бар вместо множества стадий
- Минималистичный дизайн
- Легко понять текущий статус

### ✅ Информативность
- Название текущего этапа на русском
- Номер шага (например "Шаг 3 из 5")
- Процент выполнения

### ✅ Плавность
- Анимация заполнения прогресс-бара
- Плавные переходы между значениями
- Нет резких скачков

## Стадии обработки

Компонент отображает следующие стадии:

| Backend Stage      | Отображение               | Примерный %  |
|-------------------|---------------------------|--------------|
| `initializing`    | Инициализация             | 0-10%        |
| `parsing`         | Извлечение слайдов        | 10-30%       |
| `generating_notes`| Генерация текста лекции   | 30-60%       |
| `generating_audio`| Синтез речи               | 60-80%       |
| `generating_cues` | Создание визуальных эффектов | 80-95%    |
| `completed`       | Готово                    | 100%         |

## Код компонента

### Основные фичи:

1. **Плавная анимация прогресса**
```typescript
useEffect(() => {
  const interval = setInterval(() => {
    setDisplayProgress(prev => {
      if (prev < currentProgress) {
        return Math.min(prev + 2, currentProgress);
      }
      return prev;
    });
  }, 50);
  return () => clearInterval(interval);
}, [currentProgress]);
```

2. **Подсчет шагов**
```typescript
const currentStepIndex = STAGE_ORDER.indexOf(currentStage);
const totalSteps = STAGE_ORDER.length - 1;
const currentStep = Math.max(1, currentStepIndex);
```

3. **Автоматический переход**
```typescript
useEffect(() => {
  if (currentStage === 'completed' && onComplete) {
    setTimeout(() => onComplete(), 1500);
  }
}, [currentStage, onComplete]);
```

## Props интерфейс

```typescript
interface SimpleProcessingProgressProps {
  lessonId: string;           // ID обрабатываемой лекции
  currentStage: string;       // Текущая стадия (из backend)
  currentProgress: number;    // Прогресс 0-100
  error?: string | null;      // Сообщение об ошибке (если есть)
  onComplete?: () => void;    // Callback при завершении
}
```

## Обработка ошибок

При ошибке компонент показывает:
- ❌ в заголовке
- Красный текст с описанием ошибки
- Прогресс-бар останавливается

Пример:
```
┌─────────────────────────────────────────────┐
│         ❌ Ошибка обработки                  │
│    Не удалось сгенерировать аудио           │
│                                             │
│  ████████░░░░░░░░░░░░░░░░░░  45%           │
│                                             │
│  Синтез речи • Шаг 4 из 5                  │
└─────────────────────────────────────────────┘
```

## Отладка

В режиме разработки (`NODE_ENV === 'development'`) внизу показывается ID лекции:
```
ID: a1b2c3d4...
```

Это помогает при отладке проблем с конкретными лекциями.
