## ⚙️ Что Было Исправлено

### Проблема:
- Frontend использовал пароль `admin123` для демо аккаунта
- При восстановлении БД я создал аккаунты с более безопасными паролями (`Admin123!`)
- Логин возвращал 401 Unauthorized

### Решение:
Обновил пароли в БД на простые (как ожидает frontend):
```python
admin@example.com → password: "admin123"
user@example.com  → password: "user123"
```

### Проверка:
```bash
# Тест логина
curl -X POST http://localhost:8000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@example.com","password":"admin123"}'

# Результат:
{"message":"Login successful","user":{"email":"admin@example.com","role":"admin"}} ✅
```
