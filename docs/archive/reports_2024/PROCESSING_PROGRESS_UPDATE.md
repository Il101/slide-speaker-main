# Улучшение отображения стадий обработки презентации

## Что было изменено

### 1. Создан новый компонент `ProcessingProgress.tsx`

**Путь:** `src/components/ProcessingProgress.tsx`

Этот компонент отображает детальные стадии обработки презентации с понятными описаниями:

#### Стадии обработки:
1. **Инициализация** (0%) - Подготовка к обработке файла
2. **Парсинг презентации** (20%) - Извлечение слайдов и текста
3. **Генерация лекции** (50%) - AI создает текст лекции для каждого слайда
4. **Синтез речи** (70%) - Преобразование текста в аудио
5. **Создание визуальных эффектов** (90%) - Добавление подсветок и анимаций
6. **Готово** (100%) - Презентация готова к просмотру

**Примечание:** Стадии соответствуют реальным этапам обработки на backend.

#### Особенности:
- Каждая стадия имеет свою иконку и цветовую индикацию
- Показывается общий прогресс и прогресс текущей стадии
- Анимированные переходы между стадиями
- Обработка ошибок с понятными сообщениями
- Автоматический переход к плееру при завершении

### 2. Обновлен компонент `FileUploader.tsx`

**Путь:** `src/components/FileUploader.tsx`

#### Добавленная функциональность:
- **Поллинг статуса обработки**: каждые 2 секунды проверяется статус обработки презентации
- **Интеграция с ProcessingProgress**: при переходе в стадию обработки показывается новый компонент
- **Улучшенная обработка ошибок**: показываются детальные сообщения об ошибках
- **Четкие состояния**: idle → uploading → processing → success/error

#### Состояния загрузки:
1. `idle` - Ожидание выбора файла
2. `uploading` - Загрузка файла на сервер (с прогресс-баром)
3. `processing` - Обработка презентации (показывается ProcessingProgress)
4. `success` - Успешное завершение
5. `error` - Ошибка с возможностью повторить

### 3. Обновлен API клиент `api.ts`

**Путь:** `src/lib/api.ts`

#### Добавлено:
- **Интерфейс `ProcessingStatus`**: типизация для статуса обработки
  ```typescript
  interface ProcessingStatus {
    lesson_id: string;
    status: 'processing' | 'completed' | 'failed' | 'exporting';
    progress: number;
    stage: string;
    message: string;
  }
  ```

- **Метод `getLessonStatus(lessonId: string)`**: получение текущего статуса обработки презентации
  - Путь: `GET /lessons/{lessonId}/status`
  - Возвращает детальную информацию о стадии и прогрессе

## Преимущества новой реализации

### Для пользователя:
1. **Понятно, что происходит**: каждая стадия имеет понятное описание на русском языке
2. **Видимый прогресс**: показывается как общий прогресс, так и прогресс текущей стадии
3. **Нет путаницы**: пользователь видит, почему аудио еще не доступно (оно генерируется на стадии "Синтез речи")
4. **Красивый UI**: анимации, цветовая индикация, иконки для каждой стадии
5. **Четкие ошибки**: если что-то пошло не так, пользователь видит понятное сообщение

### Технические преимущества:
1. **Отказоустойчивость**: поллинг продолжается даже при временных сбоях сети
2. **Оптимизация**: интервал поллинга 2 секунды - баланс между актуальностью и нагрузкой на сервер
3. **Типобезопасность**: все данные типизированы с помощью TypeScript
4. **Компонентный подход**: ProcessingProgress - переиспользуемый компонент
5. **Правильная очистка**: все интервалы и таймеры очищаются при размонтировании компонентов

## Что нужно на backend

Backend уже имеет необходимый endpoint `/lessons/{lesson_id}/status`, который возвращает:
- `status`: текущий статус ('processing', 'completed', 'failed')
- `stage`: текущая стадия обработки
- `progress`: прогресс в процентах (0-100)
- `message`: дополнительное сообщение

Убедитесь, что в таблице `lessons` в поле `processing_progress` обновляется информация о текущей стадии в формате:
```json
{
  "stage": "generating_audio",
  "progress": 65
}
```

## Как протестировать

1. Запустите приложение: `npm run dev`
2. Войдите в систему
3. Загрузите презентацию (PPTX или PDF)
4. Наблюдайте за процессом:
   - Сначала показывается загрузка файла
   - Затем появляется экран с детальными стадиями обработки
   - Каждая стадия меняется в соответствии с реальным прогрессом на сервере
   - После завершения автоматически открывается плеер
