## Архитектура пайплайна

```
PDF → PNG слайды → Enhanced Vision API → Гибридное распознавание → LLM → TTS → Manifest
```

### Детальный поток:
1. **PDF → PNG**: Конвертация слайдов в изображения
2. **Enhanced Vision API**: 
   - `document_text_detection` (текст + координаты)
   - `label_detection` (общие метки)
   - `object_localization` (объекты + координаты)
3. **Гибридное распознавание**: Vision API + LLM Vision для детальных описаний
4. **LLM (OpenRouter Grok)**: Генерация speaker notes и лекционного текста с учетом изображений
5. **TTS (Google)**: Синтез речи из лекционного текста
6. **Сохранение**: GCS + manifest.json

## Выявленная проблема ⚠️

### Проблема с сохранением элементов
- **Симптом**: Элементы извлекаются Enhanced Vision API, но не сохраняются в manifest.json
- **Проявление**: `"elements": []` вместо реальных элементов
- **Влияние**: LLM генерирует общий контент вместо специфичного для слайда
