# Speaker Notes Fix - SUCCESS REPORT

## Date: 2025-10-01

## Problem Solved ✅
Speaker notes were generated by LLM but saved as `null` in manifest.json

## Root Causes Found

### 1. Manifest Not Saved After Processing
**Issue**: In `backend/app/tasks.py`, the pipeline generated speaker_notes, audio, and cues, but never saved the updated manifest back to disk.

**Code Location**: `backend/app/tasks.py` lines 85-150

**Problem**:
```python
# Line 73: slides_data created from manifest
slides_data = [slide.dict() for slide in manifest.slides]

# Lines 88-94: speaker_notes added to slides_data
speaker_notes = loop.run_until_complete(...)
slide['speaker_notes'] = speaker_notes

# Lines 106-117: audio paths added to slides_data
slide['audio'] = audio_path

# Lines 136-148: cues added to slides_data
slide['cues'] = cues

# ❌ NO SAVE! Manifest on disk still has null values
```

### 2. Vision API Returning Wrong bbox Format
**Issue**: Vision API's `_get_block_bbox()` returned dict `{'x': 91, 'y': 36, 'width': 239, 'height': 75}` but Pydantic model expected list `[91, 36, 239, 75]`

**Error Message**:
```
1 validation error for SlideElement
bbox
  Input should be a valid list [type=list_type, input_value={'x': 91, 'y': 36, 'width': 239, 'height': 75}, input_type=dict]
```

**Code Location**: `backend/workers/ocr_vision.py` line 179

## Solutions Applied

### ✅ Fix 1: Save Manifest After Processing
**File**: `backend/app/tasks.py`

**Added code** after visual cues generation (line 150):
```python
# Save updated manifest with speaker_notes, audio paths, and cues
manifest_path = Path(".data") / lesson_id / "manifest.json"
manifest.slides = slides_data  # Update manifest with modified slides_data

# Convert manifest to dict and save
manifest_dict = {
    "slides": slides_data,
    "metadata": manifest.metadata if hasattr(manifest, 'metadata') else {}
}

logger.info(f"Saving updated manifest to {manifest_path}")
with open(manifest_path, 'w', encoding='utf-8') as f:
    json.dump(manifest_dict, f, ensure_ascii=False, indent=2)

logger.info(f"Manifest saved successfully with {len(slides_data)} slides")
```

### ✅ Fix 2: Convert bbox to List Format
**File**: `backend/workers/ocr_vision.py`

**Changed return type** (line 179):
```python
# Before:
def _get_block_bbox(self, block) -> Dict[str, float]:
    return {
        "x": min(x_coords),
        "y": min(y_coords),
        "width": max(x_coords) - min(x_coords),
        "height": max(y_coords) - min(y_coords)
    }

# After:
def _get_block_bbox(self, block) -> List[float]:
    x = min(x_coords)
    y = min(y_coords)
    width = max(x_coords) - min(x_coords)
    height = max(y_coords) - min(y_coords)
    return [x, y, width, height]
```

**Also fixed fallback** (line 206):
```python
# Before:
return {"x": 0, "y": 0, "width": 100, "height": 100}

# After:
return [0, 0, 100, 100]
```

**And default bbox** (line 140):
```python
# Before:
"bbox": {"x": 0, "y": 0, "width": 100, "height": 100}

# After:
"bbox": [0, 0, 100, 100]
```
