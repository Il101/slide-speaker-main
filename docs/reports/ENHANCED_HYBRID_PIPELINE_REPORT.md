## Выявленные проблемы

### 1. Проблема с русским текстом в Vision API ⚠️
Vision API неправильно распознает русский текст как цифры:
- "Анатомия листа растения" → "10000000 5000000000000"
- "Эпидермис" → "1000000000"

**Решение**: Использовать Document AI для русского текста или настроить Vision API для лучшего распознавания кириллицы.

### 2. Элементы не сохраняются в manifest ⚠️
Хотя Enhanced Vision API извлекает элементы, они не сохраняются в `manifest.json`:
- `"elements": []` вместо реальных элементов
- LLM генерирует контент на основе пустых элементов

**Решение**: Проверить процесс сохранения элементов в manifest.json.

## Архитектура пайплайна

```
1. Загрузка PDF → PNG слайды
2. Enhanced Vision API:
   - document_text_detection (текст + координаты)
   - label_detection (общие метки)
   - object_localization (объекты + координаты)
3. Hybrid Image Recognition:
   - Vision API: быстрая категоризация
   - LLM Vision: детальные описания важных изображений
4. LLM (OpenRouter Grok):
   - Генерация speaker notes с учетом текста и изображений
   - Генерация лекционного текста с упоминанием изображений
5. TTS (Google):
   - Синтез речи из лекционного текста
6. Сохранение в GCS и создание manifest.json
```
