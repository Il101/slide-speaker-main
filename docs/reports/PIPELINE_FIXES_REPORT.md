### 2. **Исправление asyncio проблем в tasks.py**
**Проблема**: `RuntimeError: Task got Future attached to a different loop`
**Решение**: Исправлен неправильный подход к работе с asyncio в Celery задачах:

**Было** (создание новых event loop'ов в каждой итерации):
```python
import asyncio
loop = asyncio.new_event_loop()
asyncio.set_event_loop(loop)
result = loop.run_until_complete(async_function())
loop.close()
```

**Стало** (использование одного event loop):
```python
import asyncio
try:
    loop = asyncio.get_event_loop()
except RuntimeError:
    loop = asyncio.new_event_loop()
    asyncio.set_event_loop(loop)

result = loop.run_until_complete(async_function())
```

**Статус**: ✅ Исправлено

### 3. **Установка Vision OCR зависимостей**
**Проблема**: `Failed to import Vision OCR provider: cannot import name 'vision' from 'google.cloud'`
**Решение**: Добавлена зависимость `google-cloud-vision==3.4.4` в `requirements.txt`
**Статус**: ✅ Исправлено

### 4. **Исправление генерации файлов ресурсов**
**Проблема**: Все запросы к `/assets/...` возвращают 404
**Решение**: 
- Исправлен путь к статическим файлам в `main.py`:
  ```python
  # Было:
  app.mount("/assets", StaticFiles(directory="/app/.data"), name="assets")
  # Стало:
  app.mount("/assets", StaticFiles(directory=str(settings.DATA_DIR)), name="assets")
  ```
- Добавлена инициализация папок данных в startup событие:
  ```python
  # Create data directories
  settings.DATA_DIR.mkdir(exist_ok=True)
  settings.UPLOAD_DIR.mkdir(exist_ok=True)
  settings.ASSETS_DIR.mkdir(exist_ok=True)
  settings.EXPORTS_DIR.mkdir(exist_ok=True)
  ```

**Статус**: ✅ Исправлено
