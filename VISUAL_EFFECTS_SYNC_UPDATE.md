### 2. Синхронизация визуальных эффектов с речью

**Проблема**: Визуальные эффекты (подсветка, подчеркивание) появлялись случайно, не синхронизированы с тем, что произносит диктор.

**Решение**: Реализована точная синхронизация с использованием SSML mark tags:

1. **LLM генерирует SSML** с метками `<mark name="word_X"/>` перед ключевыми словами
2. **Google TTS возвращает timepoints** - точное время произнесения каждой метки
3. **VisualCuesGenerator использует timepoints** для создания cues точно в момент произнесения слова

**Файлы изменены**:
- `backend/workers/visual_cues_generator.py`:
  - Добавлен новый метод `_generate_word_synchronized_cues()` 
  - Обновлена логика `generate_cues_for_slide()` для приоритета word_timings
  - Создается словарь элементов по словам для быстрого поиска
  - Highlight cue показывается в момент произнесения слова (0.8s длительность)

**Алгоритм работы**:
```python
1. LLM генерирует: "<mark name="word_растение"/>растение"
2. Google TTS возвращает: {"mark_name": "word_растение", "time_seconds": 2.5}
3. VisualCuesGenerator создает: Highlight cue для элемента со словом "растение" 
   с t0=2.5s и t1=3.3s
```

### 3. Структура данных

**word_timings в tts_words**:
```json
{
  "audio": "path/to/audio.mp3",
  "sentences": [...],
  "word_timings": [
    {
      "mark_name": "word_растение",
      "time_seconds": 2.5,
      "sentence_index": 0
    },
    ...
  ]
}
```

## Примечания

- **SSML marks критичны**: LLM должен генерировать `<mark name="word_X"/>` перед каждым важным словом
- **Fallback логика**: Если word_timings отсутствуют, используется синхронизация по предложениям
- **Производительность**: Словарь элементов создается один раз для быстрого поиска
